unicode 表情转义 收发处理

最近处理聊天内容，接收标签转码还是比较坑的，在这之前翻了下微信web版的跟去找了一些资料都是用title=‘xxx’ 描述符来转移特定的规则，
之前一项目自己也用了大概这种收发端制定一套规则来实现，但是最近做fb的项目，fb的message处理表情是直接将表情转义成unicode字符来实现的，查看了下fb的内容
fb的聊天窗口表情规则是直接带表情中带有相应的Unicode字符比如微笑() 其alt的内容直接就是一个对应的unicode字符
所以在我们客服端收到的表情的消息时接收的直接是‘xx消息’(xx为unicode字符，可以直接在网页中显示的)，因为这直接显示是很丑的，可以直接把表情的unicode字符转义成16进制(1f619)，然后有相对应的
image图片，这样的效果就比较好看了。unicode emoji字符区间 正则过滤/\ud83d[\ude00-\ude4f]/gi
